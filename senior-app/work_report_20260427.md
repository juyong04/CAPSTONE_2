# 🚀 작업 보고서: 익명 게시판 및 관리 시스템 구축 (2026.04.27)

## 🎯 1. 주요 목표
- 로그인 절차 없이 닉네임과 비밀번호만으로 작동하는 **익명 커뮤니티 게시판** 구현
- 게시판 통합 관리 인터페이스 구축 및 **작성자/관리자 권한 분리**
- IP 및 로컬 스토리지를 활용한 **동일 사용자 식별 로직** 추가
- 기존 음성 인식 글쓰기(`VoiceWriter`) 기능에 인증 플로우 연동 및 예외 처리 강화

## ✨ 2. 핵심 구현 사항

### 2.1 보안 및 사용자 식별 강화 (`utils/crypto.js`, `utils/user.js`)
- **비밀번호 암호화**: 평문 노출 방지를 위해 Web Crypto API를 활용하여 사용자가 입력한 비밀번호를 **SHA-256 해시**로 변환하여 Firestore에 저장하도록 구현했습니다.
- **IP 기반 사용자 식별**: `api.ipify.org`를 활용해 접속 IP 앞자리를 닉네임에 결합합니다. (예: `수줍은시니어(121.144)`)
- **식별자 폴백(Fallback) 로직**: 모바일 환경이나 광고 차단기 등으로 인해 IP 추출에 실패할 경우, 브라우저의 `localStorage`에 무작위 4자리 고유 식별자(예: `a1b2`)를 생성하여 동일 기기 접속 시 일관된 식별자를 유지하도록 구현했습니다.

### 2.2 권한 분리 및 UI 일원화 (`BoardPage.jsx`)
기존에 산재해 있던 관리 버튼들을 숨기고, 게시글 상세 화면 우측 상단에 **[⚙️ 글 관리]** 버튼 하나만 배치하여 UI를 깔끔하게 유지했습니다. 비밀번호 입력 결과에 따라 아래와 같이 권한이 나뉩니다.

1. **작성자 권한**: 글 작성 시 입력한 비밀번호 입력 👉 `[수정]`, `[삭제]` 가능
2. **관리자 권한**: 마스터 비밀번호 **`touch1`** 입력 👉 타인의 글이어도 `[수정]`, `[삭제]` 및 **`[📌 게시글 고정]`** 가능
3. **레거시 권한**: 예전에 작성되어 비밀번호가 없는 글의 경우 **`0000`** 입력 👉 `[수정]`, `[삭제]` 가능

### 2.3 게시글 데이터 흐름 개선 (`hooks/useBoardPosts.js`)
- **상단 고정 기능 추가**: 관리자가 글을 고정(`isPinned: true`)하면, 클라이언트 단의 정렬 로직에서 고정된 글을 항상 리스트의 최상단으로 끌어올리도록 개선했습니다.
- **레거시 데이터 호환**: 생성일(`createdAt` 또는 `date`) 필드가 없는 과거의 데이터 때문에 리스트가 렌더링되지 않던 문제를 해결하기 위해, Firestore의 쿼리 단이 아닌 **클라이언트 단에서 유연하게 정렬**하도록 로직을 수정했습니다.
- **게시판 공통 훅 적용**: `FreeBoard`, `MonthlyTopic`, `Counseling`, `Education` 4개의 모든 게시판이 인라인 Firebase 코드가 아닌 `useBoardPosts` 공통 훅을 통해 동작하도록 코드를 리팩토링 및 간결화했습니다.

### 2.4 음성 글쓰기(VoiceWriter) 연동 및 오류 방어 (`VoiceWriter.jsx`, `useBoardPosts.js`)
- **인증 플로우 추가**: 음성을 녹음하고 AI가 글을 정리해 준 직후(게시하기 직전 단계)에 **닉네임과 비밀번호 입력 창**이 나타나도록 플로우를 추가했습니다.
- **비밀번호 필수화**: 사용자가 무심코 '게시하기'를 눌러 비밀번호 없이 저장되는 것을 막기 위해, 비밀번호 입력 누락 시 알림창과 함께 저장을 차단하는 **필수 검증 로직**을 추가했습니다.
- **CORS/Storage 에러 우회 (중요)**: 로컬 개발 환경에서 Firebase Storage의 CORS 정책 차단 시 무한 대기(Hanging)에 빠지는 현상을 해결하기 위해 **5초 타임아웃(`Promise.race`)을 적용**했습니다. 음성 파일 업로드에 실패하더라도 전체 진행이 멈추지 않고 텍스트 정보는 정상적으로 DB에 저장되도록 안전망을 구축했습니다.
